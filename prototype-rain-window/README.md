# 雨の中の窓 — プロトタイプ#2

`portfolio-concept.md` のブレストから派生した、「結露」の別表現の単体プロトタイプ。
[`../prototype/`](../prototype/)(KanjiVGなぞりプロトタイプ)とは独立した、完全に別のWebGL2アプリ。

フレームワーク無し、ビルド不要。**外部アセット読み込みが一切無い**(夜景・雨は全てプロシージャル生成)ため、
`file://` で直接ブラウザに開いても動く。

## 実行方法

```sh
cd prototype-rain-window
python3 -m http.server 8080
# → http://localhost:8080 をブラウザで開く
```

## できていること

- プロシージャルに生成した夜景(ビル群のシルエット・点灯する窓・ボケた街明かり)を、
  シャープ版とガウシアンブラー版の2枚用意
- 「拭いたマスク」(蓄積+ゆっくり減衰するだけのシンプルなバッファ。流体シミュレーション不要)
  でシャープ版とブラー版をブレンド。時間とともに水蒸気でまた曇っていく
- **雨粒は個々に物理シミュレーションする本格方式**(tympanus.net/Development/RainEffect と
  webgl.souhonzan.org を参考にしたアプローチ。アセット・コードのコピーはしていない):
  - 各水滴は重力に相当する momentum を持ち、大きいほど加速しやすく、摩擦で減速する
  - 落下中は軌跡として、縮小しながら子滴を後ろに残す(トレイル)
  - 近くの水滴同士が接近すると合体する(メタボール的な見た目)
  - 各水滴は円形SDFから疑似球面レンズの法線・厚みを計算し、インスタンス描画で
    「水滴マップ」(R,G=屈折方向, B=厚み, A=アルファ)に書き込み、それを使って
    背景(シャープ版)を屈折させながら重ねる。曇っていても晴れていても、水滴の
    中だけは常にシャープに奥が見える(実際の写真の見え方と同じ)
- ページを開くと、手でガラスを拭うようなジェスチャー(大きな弧を描くスワイプ→小さな円のこすり)を
  自動再生し、その後ユーザーのドラッグ操作に引き継ぐ(プロトタイプ#1のなぞり演出と同じ設計)

## プロトタイプ#1からの教訓を反映した設計判断

- **レンダーターゲットはすべて `RGBA8` / `UNSIGNED_BYTE`。** プロトタイプ#1で
  `R16F`/`RG16F` がドライバによってレンダリング不可になり、なぞっても何も起きなくなる
  互換性問題にハマった。この曇りマスクは0〜1の値で十分なので、最初からfloat
  レンダーターゲットを一切使わない設計にして同じ問題を回避している
- **外部ファイル読み込みが無い。** KanjiVGのSVGパースで名前空間関連のパースエラーに
  ハマった教訓から、この プロトタイプは完全プロシージャルにして依存を無くした
- **メインループはtry/catchで保護**し、なぞり演出側が万一失敗しても背景・雨・拭き取りは
  動き続けるようにしてある

## チューニングのメモ

- `CONFIG.MASK_DECAY`(既定 `0.9985`/フレーム)が曇りの戻る速さ。値を下げるほど速く再び曇る
- `CONFIG.AUTOPLAY_BRUSH_RADIUS` / `USER_BRUSH_RADIUS` は `exp(-d²/radius)` のガウシアン。
  直感よりずっと小さい値が正しい(プロトタイプ#1と同じ罠)
- 夜景・窓・ボケ光は `main.js` の `sceneShader` で完結しているので、実際の作品画像に
  差し替える場合はここを画像テクスチャのサンプリングに置き換える想定
- **ブラーは低解像度FBOで行っている**(`CONFIG.BLUR_RESOLUTION`)。最初シーンと同解像度で
  ブラーしたところ、曇りがほとんど分からないほど弱かった。同じカーネルでも解像度を落として
  ブラーすることで、安いコストで強いフロストガラス感が出る
- 雨粒関連は `CONFIG.DROP_MIN_R` / `DROP_MAX_R`(粒の大きさ)、`RAIN_SPAWN_PER_SEC`(降る頻度)、
  `TRAIL_RATE`(軌跡の残しやすさ)、`MERGE_DISTANCE_FACTOR`(合体しやすさ)、
  `REFRACTION_STRENGTH`(レンズの歪み強度)で調整できる
