# 結晶 — プロトタイプ#3

`portfolio-concept.md` のブレストから派生した、レイマーチ表現の単体プロトタイプ。
[`../prototype/`](../prototype/)(KanjiVGなぞり)・[`../prototype-rain-window/`](../prototype-rain-window/)(雨の窓)とは
独立した、完全に別のWebGL2アプリ。

フレームワーク無し、ビルド不要。

## 実行方法

```sh
cd prototype-crystal
python3 -m http.server 8080
# → http://localhost:8080 をブラウザで開く
```

## できていること

- 正八面体(宝石カットの基本形)と立方体の交差(`max`)で先端を落とした「主結晶」1個に、
  同じ八面体プリミティブを縮小・オフセットしただけの副結晶4個をunion(`min`)で癒着させた
  結晶クラスターを、フルスクリーン1枚のフラグメントシェーダーでレイマーチして描画
- クラスターの外側には、`mod()`による空間のドメインリピティションで同じ結晶片プリミティブを
  実質無限に複製し、セルごとにhashでサイズ・位置を散らした「漂う結晶片」を配置。
  ポリゴンなら個別のメッシュインスタンスが要る密度を、追加ジオメトリ無しの距離場だけで
  実現している(詳細は下記「レイマーチ特有の複製表現について」)
- 法線はSDFの中心差分から求め、フレネル項で背景色(方向ベースの環境色)を反射として
  混ぜることで、真の屈折レイを飛ばさずに「ガラスっぽさ」を出している
  - さらに、冷たい主光源と別に暖色のリム光を裏側から当て、色温度のコントラストで
    「艶消しの石」ではなく宝石らしい見え方にしている
- 背景は同じレイマーチパスの中で完結する安いプロシージャル(上下グラデーション + 方向ベースの塵)
  で、外部ファイル読み込みは一切無い
- ページを開くとカメラが自動でゆっくり結晶の周りを周回し、ユーザーがドラッグした瞬間に
  自由なオービット操作へ引き継ぐ(プロトタイプ#1・#2と同じ「自動再生→ハンドオフ」の設計)
- モバイル(タッチ主体の端末)を`matchMedia("(pointer: coarse)")`で判定し、内部解像度・
  ステップ数を自動的に下げるフォールバックを最初から実装済み

## レイマーチ特有の複製表現について

最初は1個の結晶(正八面体∩立方体)だけで組んだが、「ポリゴンによる3D表示と見分けがつかない、
地味」というフィードバックを受けて、レイマーチでしか安く出せない「複製」を使って情報量を増やした。

- **試した失敗案:** `abs(p) - offset` を数回繰り返して縮小していくIFS(反復関数系)フラクタルの
  折り畳みで、1プリミティブから無限に自己相似な結晶を生やそうとした。理論上はポリゴンでは
  指数的にメッシュが増える密度を、折り畳みの反復回数ぶんの軽い演算(`abs`・比較・乗算)だけで
  出せるはずだった。実際にやってみると、offset/scaleの選び方次第で空間全体を埋め尽くす
  「無限結晶壁」になってしまい、画面いっぱいのタイル状ノイズと化した。単一の結晶が主役という
  構図が壊れる上、折り畳みの対称面に沿って暗いスジ(境界アーティファクト)も出た。
  カメラを離してもフレーミングできず(=境界が予測できない)、採用を見送った。
- **採用した方式(有限個の明示的な複製 + ドメインリピティション):**
  1. **クラスター本体**は、同じ`sdOctahedronApprox`を5回、固定オフセット・固定スケールで
     呼んで`min`するだけ。IFSと違って個数が有限なので、常にカメラのフレーミングが予測できる。
     `smooth min`は使わず素の`min`にして、鋭いファセットの継ぎ目を保ったまま演算コストも
     抑えている。
  2. **周囲を漂う結晶片**は`mod()`による正真正銘のドメインリピティションで実装。
     `mod(p + 0.5*cell, cell) - 0.5*cell`で空間そのものを繰り返し、1個のプリミティブ評価だけで
     画面内外にまたがる無数のインスタンスを配置する。セルIDに対する`hash21`でサイズと局所的な
     位置ズレを振ることで、格子の規則性が目立たないようにした。クラスター中心付近だけは
     `max(shard, keepOutRadius - length(p))`で「立入禁止球」を作り、主結晶とごちゃつかないように
     除外している。
  3. 追加コストは、クラスター側が「主結晶1回 + 副結晶4回」の`sdOctahedronApprox`呼び出し、
     結晶片側が「`mod`+hash+`sdOctahedronApprox`1回」で、単一プリミティブ版のおよそ3倍程度。
     `STEP_RELAX`は各プリミティブが引き続き安全な過小評価を返すため0.85に戻せる一方、
     `MAX_STEPS`は複数形状のunionぶん解決に必要な歩数が増えるため48〜64へ引き上げた。

## モバイル負荷対策として実際にやったこと

- **内部解像度をCSS表示サイズより落として描画し、ブラウザの拡大表示に任せる。**
  レイマーチのコストはピクセル数にほぼ比例するため、ここが一番効く。
  `CONFIG.RENDER_SCALE`(コース・ポインタ端末では`0.46`、それ以外は`0.62`)と
  `CONFIG.DPR_CAP`で内部解像度を決め、canvas要素自体はCSSで画面いっぱいに
  引き伸ばす(`width:100%; height:100%`)。追加のFBO・アップスケールパスは不要。
- **最大ステップ数(`CONFIG.MAX_STEPS`)をuniformではなくGLSLソースに直接埋め込む。**
  ループの上限に動的なuniformを使う書き方は一部のモバイルGPUドライバのGLSLコンパイラで
  最適化が効かなくなったり不安定になったりしやすいため、JSのテンプレートリテラルで
  シェーダーソースを組み立てる時点で`const`な整数リテラルにしている。
- 距離場は単純化した正八面体の近似式(8平面への距離の最小値をそのまま使う)を採用。
  厳密なSDF(IQ版、分岐あり)より軽いが、稜・頂点付近では真の距離をわずかに
  過小評価する — この過小評価は「安全な」方向(レイが面を突き抜けることはなく、
  稜付近で歩幅がやや小さくなるだけ)なので、`STEP_RELAX`で少し控えめに歩幅を取る
  だけで実用上問題ない。
- ソフトシャドウ・AO・複数バウンスの反射は実装していない。光源は1つ(主光源の
  拡散反射+スペキュラ)+ 暖色リム光のみで、いずれも追加のレイマーチを伴わない
  (法線ベクトルとライト方向の内積だけで計算できる)安い項に絞っている。
- 背景(空)も追加のレイマーチを行わない完全な解析的関数。結晶表面からの反射
  (`reflect(rd, n)`)にもそのまま使い回せるので、環境マップ用に別のテクスチャや
  パスを持つ必要が無い。

## 分かったこと・チューニングのメモ

- フレネル反射だけ(背景色を反射として混ぜるだけ)だと、背景の色数が少ないため
  「艶消しの石」のような単調な見え方になった。裏側からの暖色リム光を1項足すだけで
  色温度のコントラストが生まれ、体感の「宝石らしさ」が大きく上がった。安い実装で
  見た目の情報量を増やしたいときは、光源の色を変えるのがコスパが良い。
- チャンネル(R/G/B)ごとにフレネル指数をわずかにずらすと、エッジにごく薄い
  色収差(虹色の縁)が出て、真の分散シミュレーションをしなくても「ガラスらしさ」の
  説得力が増す。
- Playwright(Chromiumヘッドレス)でデスクトップ・モバイル双方のビューポートを
  スクリーンショットして確認。コンソールエラー無し、ドラッグ操作でのオービットと
  自動再生からのハンドオフ(ヒントのフェードアウト)も想定通り動作した。複数の
  ドラッグ角度からも継ぎ目・穴・極端な描画破綻が無いことを確認済み(ただし実機での
  fps計測はできていない。下記「次のステップ」参照)。
