# アナモルフィックな漢字 — プロトタイプ

3D空間にばらばらに散らばったガラス破片が、**たった1つの視点(スイートスポット)から
見たときだけ**漢字「水」のシルエットに整列して見える —— エイムズの部屋・強制遠近法
アート・“ありえない”チョークアートと同じ系統の単一視点アナモルフィック錯視を
WebGL2で作った単体プロトタイプ。

[`../prototype/`](../prototype/)(KanjiVGなぞり)・[`../prototype-shards/`](../prototype-shards/)
(ガラス破片)・[`../prototype-forced-perspective/`](../prototype-forced-perspective/)
(強制遠近法)などとは独立した、完全に別のWebGL2アプリ。フレームワーク無し、ビルド不要。

破片1個の見た目(正八面体のインスタンス描画・頂点ジッター・非等方スケール・自転・
`dFdx`/`dFdy`フラットシェーディング・フレネル反射/屈折・色収差・screen-space refraction)は
[`../prototype-shards/`](../prototype-shards/) の描画手法をほぼそのまま流用している。
本プロトタイプ固有の新規性は「破片の**配置**」だけにある。

## 実行方法

```sh
cd prototype-anamorphic
python3 -m http.server 8080
# → http://localhost:8080 をブラウザで開く
```

## できていること

- **単一視点アナモルフィックの核**: 各破片は「スイートスポットカメラから漢字の
  ある1点を通る3Dレイ」の上の、**ランダムな深度**に置かれている。レイ上のどこに
  置いても、そのカメラから見れば必ず同じシルエット点に射影される(深度非依存)。
  だからスイートスポットからは全破片が「水」に整列し、そこから外れると深度の
  ばらつきがそのまま見えて“ただの散らばり”に崩れる。錯視を「壊す」ための追加処理は
  一切無く、破片をカメラ向きに再配置(ビルボード化)しないだけで正しい遠近射影から
  自動的に崩れる
- **逆射影は prototype-shards の順射影の厳密な逆**。破片のスクリーン位置を決めるのは
  `uViewProj = proj * lookAt` なので、逆算にも lookAt の基底(前方 `forward=-z`、
  右 `right=normalize(up×z)`、上 `up=z×x`)を使う。シルエット点 `(px,py)` に対し
  レイ方向を
  ```
  dir = forward*FOV + right*(px*S) + up*(py*S)     // S = SILHOUETTE_SCALE
  ```
  で作る。これをビュー→射影に通すと `NDC = (px*S/aspect, py*S)` となり、深度にも
  アスペクト比にも依存しない(導出は `main.js` の `buildInstanceData` のコメントに
  詳述)。`aspect` は `NDC_x` 側にしか現れず画面の横伸びをちょうど打ち消すので、
  **ウィンドウをリサイズしても破片を作り直す必要がなく、文字は常に正方形比で表示される**
- **数値検証済み(目視ではなく計算で)**: 生成した全破片を実際の `proj*lookAt` に
  通して射影し直したところ、割り当てシルエット点との NDC 誤差は最大 `1.1e-16`
  (倍精度の丸め誤差そのもの=厳密に逆になっている)。破片は 121 個、ワールド z は
  約 `-2.94 〜 +2.20`(原点をまたいで散らばる)、スイートスポットからの距離は
  `3.49 〜 8.60`(NEAR/FAR の内側)、カメラ後方に回る破片・NDC外に出る破片ともに 0 個
- **見かけの大きさを深度で補正**(`UNIFORM_APPARENT_SIZE`)。遠近で手前の破片だけ
  巨大化して字がガタつくのを防ぐため、各破片の物理サイズを深度に比例させ、
  スイートスポットからはほぼ同じ画面サイズに見えるようにしている。別角度からは逆に
  遠い破片が物理的に巨大=散らばりが強調される副作用があり、崩れ方が派手になる
- **カメラ演出**は prototype-forced-perspective と同じ「スイートスポットで静止(第一印象=
  整列した『水』)→ 一定時間ホールド → yaw/pitch を 0 を通過する振り子運動にして
  錯視の成立/崩壊を繰り返し見せる → ドラッグで自由操作へハンドオフ」
- モバイル(`matchMedia("(pointer: coarse)")`)では内部解像度を下げ、シルエットの
  サンプリング間隔を粗くして破片数を減らす

## 対象の漢字とその作り方

- 使った漢字は **「水」**(1文字)。理由: サイトの中核コンセプト
  (`../portfolio-concept.md`「結露したガラスの向こう側」— 雨・水滴・結露・流体)に
  最も直結するモチーフであり、かつ**単純で手作業でも正しく骨格を組める**から。
  破片数を 100〜150 に収めるという制約下でも 1 文字なら各画に十分な点数を割ける
- **これはプレースホルダー**。このリポジトリにサイトオーナーの実名は含まれておらず、
  隣の [`../prototype/`](../prototype/) でも `KANJI_CODEPOINTS`(創作)に
  「実名は今後差し替え」と明記されている。実名の漢字が決まり次第、`KANJI_STROKES` を
  差し替える想定
- **フォント精度のグリフではなく「簡略化した筆画スケルトン」**。この環境には
  ブラウザ/Canvas が無く KanjiVG グリフを実際にラスタライズできないため、線分と
  2次ベジェで手打ちした 4 画(中央の縦はね+左右の払い/はらい+左上の短い払い)で
  「水」のゲシュタルトを出している(`main.js` の `KANJI_STROKES`)。各画を弧長で
  等間隔サンプリング(`SILHOUETTE_SPACING`)して正規化平面 `[-1,1]²` 上の点列にし、
  1点につき破片1個を割り当てる

## チューニングのメモ(上位定数)

`main.js` 冒頭の `CONFIG` に錯視のチューニング可能値をすべて集約:

- `SWEET_YAW / SWEET_PITCH / CAM_DIST / FOV` … スイートスポットカメラ。
  `yaw=pitch=0` で `eye=[0,0,CAM_DIST]`、原点を素直に見下ろす
- `SILHOUETTE_SCALE` … 文字の画面占有スケール(現状 0.62 で縦およそ画面高さの ~56%)
- `DEPTH_MIN / DEPTH_MAX` … レイ上の深度レンジ。**この幅が錯視の“崩れやすさ”を直接
  決める**。狭いほど別角度でも平面的に揃って見え崩れにくく、広いほど散らばって
  崩れやすい(が広すぎると手前の破片が巨大化)。現状 `3.4〜8.6`(eye が 5.6 なので
  原点前後 ±2.5 程度の層に散らばる)
- `SILHOUETTE_SPACING` … サンプリング間隔=破片数(密度)。小さいほど字が読みやすいが
  破片が増える
- `UNIFORM_APPARENT_SIZE / APPARENT_REF_DIST` … 見かけサイズの深度補正

## 視覚的に未検証な点(ブラウザで人が見て要調整の候補)

この環境では描画結果を目視できない。数式(逆射影)は上記のとおり計算で厳密に
検証したが、**「見た瞬間に読めるか / きれいか」は未検証**。人がブラウザで確認した際、
以下がまず調整対象になると予想する:

1. **「水」に読めるかどうか(最大の未検証点)**。`KANJI_STROKES` は font 精度ではなく
   手打ちの簡略骨格で、4 画の位置・角度・カーブは筆者の記憶に基づく。特に左上の短い払い
   (第3画)と左右の払いの開き角しだいで、「水」ではなく「木」「氺」的な別字に見える
   リスクがある。実際に見て、必要なら各画の制御点を微調整する
2. **サンプリング密度**(`SILHOUETTE_SPACING = 0.035 → 121 点)が字を読むのに
   最適とは限らない。粗すぎれば点線がまばらで字に見えず、細かすぎれば破片が団子状に
   重なって個々のガラス感が消える。100〜150 の範囲で振って最良点を探る必要がある
3. **深度レンジ**(`DEPTH_MIN/MAX`)。狭いと別角度でも「傾いた平らな水」に見えて
   錯視が“崩れた感”を出せず、広いと手前の破片が大きすぎて字を覆う。`UNIFORM_APPARENT_SIZE`
   をオンにしても物理サイズは深度で変わる(別角度での見え方)ので、両者のバランスは要目視
4. **破片サイズ**(`SIZE_MIN/MAX`)とシルエット点間隔の比。破片が点間隔より大きいと
   隣接破片が重なって字がベタ塗りになり、小さすぎると点が離れて字に見えない
5. **文字の縦位置**。現状シルエットの y 範囲は `-0.65〜0.90` で重心がわずかに上寄り
   (中央の縦画の上端が高い)。画面中央に正確に置きたい場合は `KANJI_STROKES` 全体を
   y 方向に約 -0.13 平行移動するか、生成時にオフセットを足す
6. **スイートスポット以外での“散らばり”の見栄え**。振り子の振幅
   (`AUTOPLAY_YAW_AMP = 0.6 ≈ 34°)で十分崩れて見えるかは要目視。崩れが弱ければ
   振幅か深度レンジを上げる
