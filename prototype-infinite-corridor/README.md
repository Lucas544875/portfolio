# 無限に沈む回廊 — プロトタイプ#5

`portfolio-concept.md` の「3. ナビゲーション:沈んでいく階層構造」を単体で確かめる
プロトタイプ。ドメインリピティションでレイマーチした窓枠が奥へ無限に後退していく回廊を作り、
潜るほど霧が濃く・暗く・冷たくなる「深度」の体感を検証する。
[`../prototype/`](../prototype/)(KanjiVGなぞり)・[`../prototype-rain-window/`](../prototype-rain-window/)(雨の窓)・
[`../prototype-shards/`](../prototype-shards/)(ガラスの破片)とは独立した、完全に別のWebGL2アプリ。

フレームワーク無し、ビルド不要。

## 実行方法

```sh
cd prototype-infinite-corridor
python3 -m http.server 8080
# → http://localhost:8080 をブラウザで開く
```

スクロール(マウスホイール)または画面の上下ドラッグで潜る深さを操作する。放置すると自動で潜っていく。

## できていること

- **z軸ドメインリピティション**で、窓枠(ラウンドボックスのリング)を z 方向に無限に繰り返す。
  `zi = mod(p.z, SPACING) - SPACING*0.5` により、どんな `p.z` も `[-SPACING/2, SPACING/2)` に
  畳み込み、各リングを SPACING の倍数の位置に**中心を合わせて**並べる。
  `-SPACING*0.5` を落とすとリングがセル端に来て鋸歯状(sawtooth)になる。この手法で最頻出のバグなので
  中心合わせを明示している。
- **窓枠リングのSDF**は「外側ラウンドボックス − 内側ラウンドボックス」。内側ボックスの z 半径を
  セルを貫通するほど大きく取り、奥まで開いた長方形の穴にすることで、カメラが枠の穴を
  くぐり抜けながら進む回廊になる。桟(枠)の太さは `FRAME_BORDER` で外側から差し引く。
- **カメラは +z へ進む**。カメラの z 位置 `ro.z` は `uDepth` uniform をそのまま使う。
  消失点は画面中央に固定したまま、カメラ位置 xy だけを微揺れ(`CAM_SWAY_*`)させてパララックスを出す。
- **フォグ/暗化は2系統を別係数で保持**する:
  - (系統1)**距離フォグ** `1 - exp(-dist*FOG_DENSITY)`。遠方の枠を霧へ溶かして消失点を作る。
  - (系統2)**深度フォグ** `DEPTH_DARKEN_MAX*(1 - exp(-uDepth*DEPTH_DARKEN_RAMP))`。潜った総量だけで
    近くの枠も含めシーン全体を暗く冷たくする。飽和カーブなので無限に潜っても発散しない。
  - 配色も深度に応じて `fogNear → fogDeep`(ほぼ暗黒の冷たい青)へ、枠自体の色温度も下げる。
- **深度連動の歪み(水圧感)**。`p.xy` に深度スケールする正弦波リップルを加えてから SDF を評価する
  (ドメインワープ)。振幅 `DISTORT_MAX*(1 - exp(-uDepth*DISTORT_RAMP))` は深部で飽和する。
  - ワープにより SDF は**厳密な距離関数でなくなる**ため、スフィアトレースのステップを
    `STEP_SAFETY`(=0.75)倍に縮めてオーバーシュート/枠の貫通(tunneling)を防いでいる。
    ワープの Lipschitz 増分は概ね `DISTORT_MAX*DISTORT_FREQ ≈ 0.07` で、厳密には 0.93 程度でも
    足りる計算だが、この環境では目視検証できないため保守的に 0.75 を採用している。
- 開くと自動で潜り、ホイール/ドラッグで操作するとハンドオフする(sibling プロトタイプと同じ
  「自動再生 → ユーザー操作へ引き継ぎ」の設計)。メインループは try/catch で保護。
- モバイル(`pointer: coarse`)では内部解像度を自動的に落とす。

## 設計判断・チューニングのメモ

- 定数はすべて `main.js` 冒頭の `CONFIG` に集約し、GLSL 側にはテンプレート展開(`f()` で
  必ず小数点付きのGLSLリテラルにする)で注入している。シェーダーとの二重管理を避けるため。
- 調整しどころ:
  - 枠の間隔・流速 → `SPACING`、`AUTO_SPEED`
  - 窓の縦横比・断面サイズ → `OUTER_HALF_X/Y/Z`、桟の太さ `FRAME_BORDER`
  - 霧の掛かり方 → `FOG_DENSITY`(距離)/ `DEPTH_DARKEN_RAMP`・`DEPTH_DARKEN_MAX`(深度)
  - 水圧の歪み → `DISTORT_MAX`(振幅上限)/`DISTORT_RAMP`(立ち上がり)/`DISTORT_FREQ`
  - 画角・望遠感 → `FOCAL`

## ★ スクロール統合フック(実サイトへ組み込むとき)

`main.js` の `depth` 変数(および uniform `uDepth`)が唯一の「潜った総量(ワールドz)」。
実サイトでは Lenis 等の慣性スクロールの `scrollY` から

```js
depth = scrollY * PIXELS_TO_DEPTH;
```

の1行で置き換わるように、深度入力を1変数へ集約してある。単体プロトタイプの auto-advance と
ホイール/ドラッグは、その `scrollY` が無い環境用の代替入力。フラグメントシェーダーの
`uDepth` を受ける箇所(カメラ `ro.z`、フォグ、歪み)にも同じ趣旨のコメントを置いた。

## ★ ホラー演出フック(最下層だけ影がよぎる)

`portfolio-concept.md`「最下層(Contact等)だけ、ガラスの向こうを一瞬何かの影がよぎる」用の
スロットを意図的に空けてある。

- 定数 `CONFIG.HORROR_DEPTH_THRESHOLD`(既定 220.0 ワールドz)= 最下層とみなす深度。
- フラグメントシェーダー末尾の `if (uDepth > HORROR_DEPTH_THRESHOLD) { ... }` ブロックが実装場所。
  現状は影を差すコード例をコメントアウトしたスタブ。将来案:別SDF(人影/手)を消失点付近だけに
  極薄アルファで一瞬合成する、等。**「静かな不穏さ」を壊さないよう、出現は極短時間・低頻度・
  低コントラストに留めること**をコメントに明記している。

## 視覚的に未検証な点

この環境にはヘッドレスブラウザ/スクリーンショットが無く、**WebGL出力を目視できていない**。
数式(mod中心合わせ、リングSDFの外−内、ステップ安全率のLipschitz見積り)は手計算で確認済みだが、
実際にブラウザで見たとき最初に調整が要りそうなのは以下:

- **`AUTO_SPEED`(自動で潜る速さ)は完全に当て推量**。枠が速すぎ/遅すぎる可能性が高い。
- **フォグ曲線の急峻さ(`FOG_DENSITY` / `DEPTH_DARKEN_RAMP` / `DEPTH_DARKEN_MAX`)は未検証**。
  暗くなりすぎて何も見えない、または逆に全然沈んで見えない、のどちらかに振れやすい。
- **窓枠の断面比・桟の太さ(`OUTER_HALF_*` / `FRAME_BORDER`)** が画角(`FOCAL`)と噛み合っているか未確認。
  枠が画面外に大きすぎる/穴が小さすぎて閉塞して見える、といったズレはブラウザで一発調整が必要。
- **`STEP_SAFETY`=0.75 は保守側に倒した値**。実際に貫通が出なければ 0.85〜0.9 まで上げて
  マーチ回数を減らせる(パフォーマンス改善余地)。逆に `DISTORT_MAX` を上げるなら安全率も要見直し。
- **`DISTORT_MAX`(歪み振幅)** が「不穏な揺らぎ」として効くか、単にブレて見えるだけかは要目視。
- `MAX_STEPS`=96 / `RENDER_SCALE` が実機のフレームレートで足りるかは未計測。
