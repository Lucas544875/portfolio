# ヒーロー・プロトタイプ:「結露したガラスの向こう側」

[`portfolio-concept.md`](../portfolio-concept.md) の「次の一歩」で挙げた、
ヒーローセクション単体プロトタイプ(結露ガラス + KanjiVG拭き文字)の実装。

フレームワーク無し、WebGL2 + Vanilla JS のみ。ビルド不要。

## 実行方法

`fetch` で KanjiVG の SVG を読み込むため、`file://` では動かない。ローカルサーバーが必要。

```sh
cd prototype
python3 -m http.server 8080
# → http://localhost:8080 をブラウザで開く
```

## できていること

- WebGL2 上に stable-fluids 風の流体シミュレーション(移流・粘性渦・圧力投影)を実装し、
  「結露で曇ったガラス」を dye テクスチャの曇り度として表現
- ページを開くと、KanjiVG のストロークパスデータ(`assets/kanjivg/`)を正しい筆順で
  自動的になぞり、なぞった軌跡だけガラスの曇りが拭われて奥の景色が見える
- なぞり終わった後(またはユーザーが操作した瞬間)、マウス/タッチのドラッグで
  自由にガラスを拭える。渦や乱流が発生し、曇りは時間とともにゆっくり戻る
- 背景の「奥の景色」はプレースホルダーのプロシージャルなグロー(実際の作品画像に差し替え予定)

## 使用している漢字について

「創作」(そうさく)をプレースホルダーとして使用(git上のユーザー名が確認できなかったため)。
実際の名前の漢字に差し替える場合は `main.js` の `KANJI_CODEPOINTS` を、対応する
Unicode コードポイント(5桁16進、例: `創` → `05275`)に変更し、対応するSVGを
[KanjiVG](https://github.com/KanjiVG/kanjivg/tree/master/kanji) から
`assets/kanjivg/` に配置する。

## 分かったこと・チューニングのメモ

- **なぞり跡に注入する速度(velocity)を大きくしすぎると、渦の乱流で文字がすぐ
  読めなくなる。** `CONFIG.AUTOPLAY_VELOCITY_SCALE` を自由拭き用の速度
  (`SPLAT_FORCE`)よりずっと小さくすることで、直後の文字形状の可読性を保っている。
  それでも数秒経つと初期のストロークから徐々に渦で崩れていく — これは意図せず
  「発見」した効果だが、「ガラスに書いた文字がじわっと滲んでいく」という
  ホラー演出の伏線としても使えそう。
- 曇り(dye)の半径は UV 空間で `exp(-d²/radius)` のガウシアン。`radius` は直感より
  ずっと小さい値が正しい(例: 太い手拭きは `0.018` 前後、細い筆跡は `0.00012` 前後)。
- モバイル対応、実際の作品画像への差し替え、レスポンシブなレイアウト調整は未着手。
  まずは「手応え」の確認が目的のプロトタイプ。
